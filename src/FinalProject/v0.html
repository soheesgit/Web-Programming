본인 역량에 대한 자유로운 기술

저는 문제 해결 능력과 끈기를 가장 큰 강점으로 생각합니다. 어려운 문제를 만났을 때 포기하지 않고 다양한 방법으로 접근하여 해결책을 찾아내는 것을 좋아합니다. 이러한 성향은 프로그래밍을 할 때 특히 큰 도움이 됩니다.

또한 새로운 기술을 배우는 것에 대한 열정이 있습니다. 기술 트렌드를 꾸준히 따라가며 필요한 기술을 습득하는 데 주저하지 않습니다. 최근에는 Spring Boot와 JPA를 중점적으로 학습하고 있으며, 이를 활용한 프로젝트를 진행하고 있습니다.

팀 프로젝트에서는 소통과 협업을 중요시합니다. 명확한 의사소통을 통해 팀원들과 효율적으로 협업하며, 필요할 때는 리더십을 발휘하여 프로젝트를 성공적으로 이끌어 나갑니다.

기술적 도전

프로젝트 중 어려웠던 부분과 해결 방안

데이터베이스 연동 문제

프로젝트 진행 중 발생한 DB 연동 이슈

문제 상황

팀 프로젝트에서 Spring Boot 애플리케이션과 MySQL 데이터베이스를 연동하는 과정에서 JPA 엔티티 매핑 오류가 발생했습니다. 특히 다대다 관계를 설정하는 과정에서 무한 루프 참조 문제가 발생하여 API 응답이 제대로 반환되지 않았습니다.

해결 방안

1. 엔티티 클래스에 @JsonManagedReference와 @JsonBackReference 어노테이션을 적용하여 직렬화 과정에서의 순환 참조를 해결했습니다.
2. 다대다 관계를 중간 테이블을 이용한 일대다, 다대일 관계로 재설계했습니다.
3. DTO 패턴을 도입하여 엔티티와 API 응답을 분리함으로써 필요한 데이터만 클라이언트에 전달하도록 개선했습니다.

배운 점

이 문제를 해결하면서 JPA의 엔티티 관계 설계의 중요성과 API 응답 설계 시 고려해야 할 사항들에 대해 깊이 이해하게 되었습니다. 또한 문제 해결을 위해 다양한 자료를 찾아보고 팀원들과 함께 고민하는 과정에서 협업의 중요성도 다시 한번 느낄 수 있었습니다.

서버 성능 최적화 도전

대용량 데이터 처리 시 발생한 성능 이슈

문제 상황

사용자 데이터가 증가함에 따라 특정 API 엔드포인트의 응답 시간이 크게 늘어나는 문제가 발생했습니다. 특히 여러 테이블을 조인하여 데이터를 조회하는 API에서 성능 저하가 두드러졌습니다.

해결 방안

1. 데이터베이스 인덱스를 적절히 추가하여 쿼리 성능을 개선했습니다.
2. N+1 문제를 해결하기 위해 JPA의 fetch join을 활용했습니다.
3. 자주 조회되는 데이터에 대해 Redis 캐싱을 도입하여 데이터베이스 부하를 줄였습니다.
4. 페이지네이션을 적용하여 한 번에 처리하는 데이터의 양을 제한했습니다.

배운 점

이 경험을 통해 대용량 데이터 처리 시 성능 최적화의 중요성을 깨달았습니다. 단순히 기능이 작동하는 것을 넘어, 확장성과 성능을 고려한 설계의 필요성을 체감했습니다. 또한 문제 해결 과정에서 다양한 성능 최적화 기법들을 학습하고 적용해볼 수 있었던 값진 경험이었습니다.

AI가 대체 가능한 업무에 대한 채용 감소 이슈

뉴스 기사 요약

최근 '카카오가 AI가 대체 가능한 직무에 대해 채용을 줄이겠다'는 보도가 있었습니다. 비록 해당 기업은 이를 부인했지만, 이는 AI 기술의 발전으로 인한 일자리 변화에 대한 사회적 논의를 불러일으켰습니다. 국내 IT 기업들이 AI로 대체할 수 있는 신입·저숙련 개발자 고용을 줄이는 추세가 나타나고 있다는 내용이었습니다.

최근 'AI가 대체 가능한 직무에 대해 채용을 줄이겠다'는 보도를 접하며 깊은 고민을 하게 되었습니다. 비록 보도된 기업은 이를 부인했지만, 저는 기본적이고 반복적인 업무는 이미 AI에 의해 빠르게 대체되고 있다고 느꼈습니다. 실제로 자연어처리, 자동 코드 생성, 상담 챗봇 등 여러 기술들이 일상화되면서, '어떤 일을 사람이 해야 하는가'에 대한 기준이 급변하고 있다고 생각합니다.

저는 'AI가 모든 직무를 대체할 수 있다'는 주장에는 회의적입니다. 그러나 'AI로 인해 인간이 투입하는 시간이 줄어들면서, 조직이 필요한 인력을 줄여가는 것'은 충분히 납득 가능한 흐름이라 봅니다. 이는 비효율을 줄이기 위한 방향일 뿐, 인간의 가치를 무시하려는 것이 아니기 때문입니다.

하지만 동시에, AI의 발전 속도는 무서우리만치 빠르다는 사실도 절감합니다. 기술적 한계가 여전히 존재하지만, 앞으로 10년 안에 고도화된 AI가 더 많은 분야에서 사람의 자리를 위협할 수 있다는 생각에 경각심을 갖게 됩니다. 이런 흐름 속에서 저는 기술의 발전을 수용하는 동시에, 인간만의 경쟁력을 키우기 위해 끊임없이 배워야 한다고 느꼈습니다. 이러한 고민과 각성은 제가 기술을 대하는 태도에도 긍정적인 영향을 주고 있습니다.

제가 가진 직업관은 '기술은 도구일 뿐이며, 그 도구를 어떻게 활용하느냐에 따라 인간의 가치는 오히려 더욱 커질 수 있다'는 믿음에서 출발합니다. 최근 AI 기술이 빠르게 확산되면서 '사람이 필요 없는 시대'가 올 수 있다는 불안도 커지고 있지만, 저는 이러한 변화가 오히려 사람의 고유한 역량을 더 명확하게 드러내는 기회라고 생각합니다.

AI가 단순하고 반복적인 업무를 대신해주는 시대에는 창의력, 문제해결능력, 공감 능력 등 인간적인 요소들이 더 중요해집니다. 따라서 저는 기술을 익히는 데 그치지 않고, 기술의 작동 원리와 인간 중심의 적용 방법까지 함께 고민하려고 노력합니다.

앞으로 저는 개발자로서 단순히 코딩을 잘하는 것을 넘어, 기술을 윤리적이고 인간 친화적으로 설계할 수 있는 사람이 되고 싶습니다. AI가 기술적으로 완전해질수록, 그 기술이 인간 사회에 어떻게 연결되고 활용되는지가 더욱 중요한 이슈가 될 것이라 믿기 때문입니다.
