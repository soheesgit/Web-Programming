컴퓨터와의 첫 만남

▲ 중학교 정보 시간에 사용한 스크래치(Scratch)

처음부터 '개발자'를 꿈꾼 건 아니였습니다. 저는 단순히 컴퓨터를 좋아하는 아이에 가까웠습니다. 학원에 다니거나 특별한 프로젝트를 해본 적도 없고, 코딩이라는 단어조차 익숙하지 않았습니다.

"컴퓨터를 다룰 때는 몇 시간이 순식간에 지나갈 만큼 몰입할 수 있었고, 시간 감각이 무뎌질 정도로 집중했습니다."

저에게 있어 프로그래밍을 처음 접한 경험은 중학교 2학년 정보 시간이었습니다. 그때는 본격적인 코딩 언어가 아닌, 스크래치(Scratch)라는 프로그램을 사용한 수업이었습니다. 스크래치는 화면 속 고양이 캐릭터에게 어떤 동작을 하게 만들지 명령을 조합해 게임이나 간단한 프로그램입니다.

핵심 깨달음

컴퓨터에게 명령을 내려 원하는 결과를 만드는 과정 자체에 흥미를 느꼈습니다.

2023년, 1학년 1학기

첫 프로그래밍 경험

▲ 첫 프로그래밍 경험 - Hello World

대학에 입학하면서 처음으로 프로그래밍을 배우게 되었습니다. 입학 전까지는 단 한 줄의 코드도 작성해본 적이 없었습니다. C프로그래밍 수업 첫 시간, 화면에 "Hello World!"를 띄우기 위한 코드를 따라 입력했을 때, 저는 예상보다 훨씬 많은 어려움을 겪었습니다.

hello.c 
#include <stdio.h>
int main() {
    printf("Hello World!");
    return 0;
}

분명 교수님이 작성한 코드와 똑같이 썼다고 생각했지만, 제 화면에서는 실행이 되지 않았습니다. 에러 메시지만 화면에 출력됐고, 당황한 저는 반복해서 코드를 지우고 다시 쓰기를 몇 차례 반복했습니다. 알고 보니 #include <stdio.h>를 #include <studio.h>로 작성한 것이 원인이었습니다.

그 한 줄을 제대로 실행시키는 데 무려 1시간이 걸렸습니다. 하지만 신기하게도, 짜증이 나거나 포기하고 싶지는 않았습니다.

2023년, 1학년 2학기

신규채용 대신 AI로 카카오의 인사 혁신

2025.04.17

AI가 대체가능한 업무에 사람 안뽑고 자리도 없애 고용 불안 현실화 우려

카카오가 코딩 등 인공지능(AI)이 대신할 수 있는 직무는 신규 채용을 제한하기로 했다. 생성형 AI가 업무 생산성 향상에 활용되는 것에서 나아가 사람 자체를 대체하는 'AI발 일자리 충격'이 현실화하고 있다.

"한국은 인력 효율화가 쉽지 않기 때문에 채용 때부터 AI가 대신할 수 있는지를 중요시할 수밖에 없다"며 "AI 기술이 고도화될수록 고용 안정성이 흔들리는 직군이 늘어날 것"

- 판교의 한 IT기업 최고기술책임자(CTO)

17일 카카오 복수의 내부 관계자들에 따르면 카카오는 이달 초 인력 운용 관련 설명자료를 사내 게시판에 게재했다. 소프트웨어 개발 등 현업에서 AI를 접목해 효율성을 높이자는 것이 주된 내용으로, 신입 개발자 대신 AI가 할 수 있는 업무들을 열거하고 실제로 AI로 인력을 대체하고 있는 업계 사례가 참고용으로 제시됐다.

카카오 공식 입장

2025.05.18

카카오가 코딩 등 인공지능이 대신할 수 있는 직무에 대해 신규 채용을 제한하기 위해 인력 운용 설명 자료를 사내 게시판에 게재했다는 보도내용은 사실이 아닙니다. 카카오는 이러한 취지의 자료를 작성하거나 공지를 진행한 바 없으며, 관련 계획도 없음을 알려드립니다.

IT 개발 직무 신규 채용 공고 추이

출처: 진학사 캐치 (2023-2025)

내 생각

문소희 2025.05.27

나는 평소에도 챗GPT를 자주 사용한다. 단순한 문장 생성뿐만 아니라 코드 예제, 오류 해결, 특정 기능 구현 등에서 GPT는 놀라울 만큼 높은 정확도를 보여준다.

실제 경험

내가 공부 중 에러가 나는 코드를 프롬포트에 붙여넣자 GPT가 문제점을 빠르게 찾아내고, 고쳐주는 경험을 했을 땐 정말 충격을 받았다. 제시한 코드는 완벽하게 작동했고, 그 순간 'AI가 개발 업무의 상당 부분을 대체할 수도 있겠다'는 위기감이 현실처럼 다가왔다.

이런 경험이 쌓이면서, 친구들과도 "이러다 진짜 개발자 안 뽑는 거 아니야?"라는 이야기를 자주 나누게 되었다. 이제는 그 말이 단순한 농담으로만 들리지 않는다.

이번 카카오 뉴스는 내가 막연하게 느껴오던 두려움과 위기의식을 한 번에 자극한 사건이었다.

나는 앞으로 AI가 지금보다 훨씬 더 빠르고 강력한 속도로 발전해나갈 것이라고 생각한다. 몇 년 전까지만 해도 상상하지 못했던 일들이 이제는 눈앞에서 실현되고 있고, AI는 그 속도를 늦출 생각이 없어 보인다.

미래 예측

  • 단순 반복 업무는 AI가 대부분 대체
  • 사람은 문제를 정의하고 방향을 설정하는 역할에 집중
  • 기업마다 소수의 '기술의 본질을 이해하고 혁신을 주도하는 상위 1% 개발자'들만 남게 될 것

AI는 이미 효율성과 생산성 측면에서 사람을 능가하는 부분이 많고, 내가 기업이여도 주니어 개발자보다는 AI를 믿고 쓸 수 있겠다는 생각이 들었다.

이번 뉴스는, 나에게 "AI 시대에 인간 개발자로서 어떤 가치를 가져야 하는가"라는 본질적인 질문을 던지게 만드는 계기였다. 특히 신입 개발자, 주니어 개발자 입장에서 어떤 역량을 갖추고 있어야 AI에게 밀리지 않고 살아남을 수 있을지 진지하게 고민해보는 계기가 되었다.

나는 앞으로 AI가 쉽게 대체할 수 없는 역량, 특히 문제를 정의하고 구조적으로 해결하는 능력을 갖추기 위해 집중적으로 노력하고 있다.

나의 전략

01
문제 정의 능력 강화

단순히 기능을 구현하는 데 그치지 않고, 사용자 입장에서 이 기능이 정말 필요한가, 더 효율적인 구조는 무엇인가를 먼저 고민

02
AI를 도구로 활용

AI를 위협으로 받아들이기보다는 도구로써 내가 주도적으로 활용해야 할 대상으로 인식

03
능동적 학습과 응용

GPT가 제안한 코드는 단순히 정답이 아니라, 하나의 선택지일 뿐. 그 원리를 파악하고, 필요하면 내 프로젝트에 맞게 구조를 바꾸거나 완전히 재설계

나는 앞으로도 변화를 두려워하지 않고, 기술을 내 편으로 삼아 성장할 수 있는 개발자, 그 중심에 AI를 능동적으로 활용할 줄 아는 개발자로 나아갈 것이다.

프로젝트 1. 테트리스

프로젝트 개요

1학년 C프로그래밍 실습 수업에서 2인 1조로 진행한 팀 프로젝트입니다. C언어를 활용해 기존 테트리스의 기본 틀에 다양한 기능을 직접 설계하고 구현하였으며, 약 한 달간 랜덤으로 매칭된 팀원과 함께 기획, 개발, 디버깅 전 과정을 수행했습니다.

주요 기능 구현 내용

점수 시스템
  • 한 줄을 제거할 때마다 기본 점수가 부여되며, 한 번에 여러 줄을 제거할 경우 가중치가 적용되어 더 높은 점수 획득
  • 누적 점수를 바탕으로 게임 종료 후 리더보드에 점수를 저장하고, 상위 순위는 화면에 표시
플레이타임 및 속도 조절
  • 실시간으로 플레이 시간을 측정해 화면에 표시

02
일시정지 중 플레이타임 계산 오류
문제점

'P' 버튼을 누르면 일시정지되도록 하는 기능은 어렵지 않게 구현했지만, 문제는 플레이 시간이었습니다. 일시정지 중 시간까지 플레이타임에 포함되는 오류가 발생해, 사용자가 오랜 시간 정지해도 플레이타임이 계속 증가하는 문제가 생겼습니다.

해결 방법

'P'를 누른 시점과 해제된 시점을 기록한 뒤, 해당 시간을 전체 시간에서 제외하는 방식으로 정확한 플레이타임 측정을 구현했습니다.

pause_time.c 
// 일시정지 시간 계산 코드
if (isPaused) {
    pauseStartTime = currentTime;
} else {
    totalPauseTime += (currentTime - pauseStartTime);
}
actualPlayTime = totalTime - totalPauseTime;

문제점

typedef struct {

// 파일에 구조체 단위로 저장
fwrite(&playerRank, sizeof(PlayerRank), 1, fp);
04
블록 색상 지정 시 전체 화면 색 변경 오류
문제점

초기에는 블록에 색을 입히는 과정에서 전체 콘솔 색이 바뀌는 문제가 발생했습니다.

해결 방법

블록마다 색상을 개별적으로 지정하기 위한 방식이 필요했고, 여러 번 시도 끝에 블록을 구조체로 정의하고, `exist` 요소와 함께 `color` 요소를 추가하여 각 블록마다 고유 색을 가지도록 구조화하는 방식으로 문제를 해결했습니다.

block_color.c 
typedef struct {
    int exist;  // 블록 존재 여부
    int color;  // 블록 색상 코드
} Block;

// 색상 적용 함수
void drawBlock(Block block) {
    if (block.exist) {
        SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), block.color);
        printf("■");
        SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), 7); // 기본색 복원
    }
}

@Entity(foreignKeys = @ForeignKey(

    parentColumns = "id",
    childColumns = "plantId",
    onDelete = CASCADE

public class Routine {

NotificationHelper.java 
// Android 13 이상에서 알림 권한 요청
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    if (ContextCompat.checkSelfPermission(this,
            Manifest.permission.POST_NOTIFICATIONS) !=
            PackageManager.PERMISSION_GRANTED) {

        ActivityCompat.requestPermissions(this,
            new String[]{Manifest.permission.POST_NOTIFICATIONS},
            NOTIFICATION_PERMISSION_CODE);
    }
}

// 알림 채널 생성
private void createNotificationChannel() {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
        NotificationChannel channel = new NotificationChannel(
            CHANNEL_ID,
            "Plant Care Reminders",
            NotificationManager.IMPORTANCE_DEFAULT);
        channel.setDescription("Reminders for plant care routines");

        NotificationManager notificationManager =
            getSystemService(NotificationManager.class);
        notificationManager.createNotificationChannel(channel);
    }
}

Room 쿼리 결과 null 처리 미흡으로 인한 앱 강제 종료
문제점

Routine이나 History 조회 쿼리에서 결과가 null이거나 비어 있을 때 이를 처리하지 못해, 앱이 크래시되는 문제가 발생했습니다. 특히 LiveData를 통해 RecyclerView에 데이터를 바인딩할 때, 리스트가 null인 상태에서 submitList를 호출하면서 `NullPointerException`이 발생했습니다.

해결 방법

DAO 반환값을 `List<>`로 고정하고, ViewModel에서 `?: emptyList()`를 사용해 방어 코드를 추가하여 문제를 해결했습니다.

PlantViewModel.java 
// DAO 메서드
@Query("SELECT * FROM routine WHERE plantId = :plantId")
LiveData> getRoutinesByPlantId(long plantId);

// ViewModel에서 방어 코드 추가
private val _routines = MediatorLiveData>()
val routines: LiveData> = _routines

// 데이터 처리 시 null 방어
routineDao.getRoutinesByPlantId(plantId).observe(owner) { result ->
    _routines.value = result ?: emptyList()
}

// Adapter 업데이트 시
adapter.submitList(routines.value ?: emptyList())
